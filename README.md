# pch-corvette

An interactive road trip journal documenting a 2021 Pacific Coast Highway drive from Astoria, OR to San Diego, CA, returning via Nevada and I-5. Built around a full-screen map where users can click anywhere along the route to stream dashcam footage from that exact moment in the drive, with starred POI markers that open photo galleries for major stops.

---

## What This Is

This is a statically-hosted, media-heavy personal site. There is no traditional backend — the app is a Next.js static export served from Cloudflare Pages, with media assets (video clips, photos) stored in Cloudflare R2. All route logic (GPS → video file + timestamp offset resolution) runs client-side against a pre-built JSON index generated by an offline Python data pipeline.

### Data Pipeline (`pipeline/`)

An offline, one-time (or infrequent) Python pipeline that ingests raw data sources and produces static artifacts consumed by the frontend.

**Inputs:**
- PDR MP4 files from the C8's Performance Data Recorder (SD card)
- `Timeline.json` from Google Maps Timeline export (authoritative GPS route)
- Photos with EXIF geotags
- Scanned keepsake images (manually tagged)

**Outputs:**
- `route.geojson` — the full trip route as a GeoJSON `LineString` for MapLibre
- `clip_index.json` — maps GPS coordinate ranges to `{ filename, timestamp_offset_s }` for each PDR clip
- `pois.json` — POI metadata: coordinates, name, photo asset list, description
- Transcoded video files — PDR clips re-encoded to H.264 MP4 at web bitrate, with HLS manifests for adaptive streaming

**Key pipeline steps:**

1. **GPS extraction** — Run `exiftool -ee` against PDR MP4s to extract embedded GPS streams. Fall back to Cosworth Toolbox CSV export if the GPS data is in a proprietary binary track. Merge with Google Timeline data, using Timeline as the authoritative source for the overall route and PDR GPS for precise per-clip timestamp alignment.

2. **Route construction** — Parse Google Timeline's `semanticSegments[].timelinePath` into a deduplicated, chronologically ordered array of `(lat, lng, timestamp)` tuples. Serialize to GeoJSON.

3. **Clip index construction** — For each PDR clip: extract its GPS start/end coordinates and start timestamp. Build a spatial index (simple sorted array for a trip of this scale) mapping any `(lat, lng)` point on the route to the clip that contains it and the approximate seek offset in seconds.

4. **Transcoding** — FFmpeg pipeline: input 1080p PDR MP4 → output H.264 MP4 at ~4Mbps for direct playback + HLS `.m3u8` manifest for adaptive bitrate. Originals archived in a separate R2 prefix.

5. **Photo processing** — Extract EXIF GPS from trip photos; cluster by proximity to POIs; generate thumbnails via Pillow.

### Frontend (`web/`)

**Stack:** Next.js (static export) · TypeScript · MapLibre GL JS · Tailwind CSS · shadcn/ui · Framer Motion

**Map layer (MapLibre GL JS + MapTiler basemap):**
- Renders `route.geojson` as a clickable `LineString` layer
- On route click: resolves the clicked `LngLat` against `clip_index.json` using nearest-point lookup, loads the corresponding video URL from R2, and opens the `VideoModal` seeked to the correct offset
- POI markers rendered as custom SVG star elements; click opens `PhotoGallery` modal

**Component breakdown:**
- `RouteMap` — full-screen MapLibre map, manages click handlers and modal state
- `VideoModal` — Plyr-based HTML5 video player, accepts `{ src, startTime }` props; streams HLS from R2
- `POIMarker` — custom MapLibre marker component with hover/click states
- `PhotoGallery` — lightbox modal for photos and scanned keepsakes at a given POI
- `TimelineBar` — optional bottom-of-screen scrubber showing the full route with temporal position indicator

**Client-side GPS resolution:**
```
userClick(lngLat)
  → nearestPointOnLine(lngLat, route.geojson)     // turf.js
  → binarySearch(clip_index.json, normalizedOffset)
  → { clipUrl, seekSeconds }
  → VideoModal.open({ src: R2_URL + clipUrl, startTime: seekSeconds })
```
No network requests at interaction time beyond streaming the video itself.

## Development Phases

**Phase 1 — Pipeline foundation**
Validate GPS extraction from PDR MP4s via `exiftool`. Merge with Google Timeline export. Produce `route.geojson` and a working `clip_index.json` against a small subset of footage. Verify the coordinate → clip → seek offset lookup is accurate end-to-end.

**Phase 2 — Map prototype**
Next.js app skeleton. MapLibre rendering `route.geojson` with MapTiler basemap. Hardcoded POI markers. No video yet.

**Phase 3 — Video integration**
Wire up the click → index lookup → VideoModal flow. Upload a handful of transcoded clips to R2. Confirm seek accuracy.

**Phase 4 — Photos & POIs**
Populate `pois.json`. Build `PhotoGallery` modal. Process and upload all trip photos and scanned keepsakes.

**Phase 5 — Transcoding & full upload**
Run FFmpeg pipeline against all PDR footage. Upload to R2. Final `clip_index.json` covering the full trip.

**Phase 6 — Polish & deploy**
Visual design pass, Framer Motion transitions, mobile responsiveness, Cloudflare Pages deployment.

## Known Facts & Open Questions

- **PDR GPS format**: Confirmed — the PDR MP4s contain a rich binary telemetry data stream alongside video (GPS coordinates, speed, heading, g-force, brake/throttle). A fix is in progress to ensure this data stream is preserved when the source files are segmented into normalized chunks.
- **Timeline data completeness**: Confirmed — Google Timeline export has continuous semantic segment coverage for May 29–June 13, 2021 (402 segments). `timeline_trip_excerpt.json` is the authoritative GPS source for route construction.
- **Timestamp alignment**: PDR clips carry a UTC `creation_time` in their MP4 metadata; Timeline GPS points also carry UTC timestamps. Time-based alignment is the primary method for mapping clips to route positions.
- **Video segment gaps**: PDR auto-segmentation creates variable-length clips (normalized to 5-min chunks). Need to handle edge cases where a clicked map point falls between two clips — show nearest clip or seek to end of prior clip.
